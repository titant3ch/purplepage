Purpose:

 

This document is to guide the VSSC Agents on how to Complete/Terminate PIDs.

 

Criteria to Complete the PID.

 

When the Customer has spoken to or emailed the VSSC in reference to the PID request.

 

 

Steps to Complete a PID:

 

1.      VSSC enters detailed comments in Nexus notes per current process

 

2.      VSSC will Complete a request only using one of following applicable Completed Reason Codes from the list below:

Completed

Generic-No other category applies-Consulting/Contacted a Customer

Completed/Feedback to CTS

If feedback to CTS is required this will over-ride any other code

SSC Completed .com

All .com solutions including Flat File Tool

SSC Completed SWO

 

SSC Completed Multi

More than one solution was addressed

SSC Completed FXCT

 

 

3.      From the Information Tab select the Complete Request button

4.      Completed is the Only Option that should be selected

5.      Select the "Save record" icon from the Toolbar menu.

 

*Note: Once a request has been 'Completed' it cannot be uncompleted.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 


Steps to Terminate a PID:

 

1.      Detailed comments are entered in Nexus notes per the current process.

2.      From the Information Tab select the Terminate Request button.

3.      The system immediately displays the following message prompting for a Terminated Reason.


4.      Select the appropriate Terminated Reason from the drop-down choice list indicated below.

5.      Save

 

 

Although the drop down menu will display a full list of Terminate Reasons, PIDs Should Only be Terminated with the Following Reasons:

 

 

 

 

 

 

 

 

 

 

 

Unavailable/Called 3 times:    If the VSSC is unsuccessful in reaching the customer after a minimum of 2 attempts on 2 different days the request should be terminated.  Leaving a Voicemail or message for the contact person does not constitute a successful contact with the Customer.

 

Note: VSSC will select Unavailable/Called 3 times but the expectation is for this reason code to be utilized when Customer was Unavailable/Called 2 times per our current process.

 

Duplicate Request:                 When a duplicate request is received.

 

 

Sales Cancelled:                    This is used if the AE contacts the VSSC to cancel the request prior to the VSSC having a conversation with the Customer.  The PID would be terminated.

 

 

 

 

 

 

Nexus will only allow the VSSC to terminate a Request when the Request Status is Pending CA Action and the Pioneer Request ID is not Null.

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 


Steps to Un-Terminate a PID:

 

 

Technically a PID can be Un-Terminated within 14 days after the initial termination.

VSSC will use the Un-Terminated option when the original PID was terminated and the Customer calls in within 14 business days after the fact for additional assistance.

 

*Please note the VSSC that terminates the PID is the only one that can un-terminate. While viewing the PID you will be able to see who the VSSC was by reviewing the Transfer to CIC field.

 

See screen shot below on how to Un-Terminate a PID request. Nexus notes must be updated indicating reason for Un-Terminating a request.

 

 

 

 

 

If a FSSC receives a request from the AE/Customer to Un-Terminate a request and complete Training within 14 Days after the initial terminated PID.

 

FSSC will:

 

1.      Update Nexus Notes on the PID indicating training needs.

2.      Email the VSSC who terminated the PID to Un-terminate and schedule the Customers Training.

 

VSSC will:

3.      Upon receiving the FSSCs email, VSSC will extract request from within their workload (note: Request will already be in VSSC workload as the original PID was terminated by VSSC).

 

4.      Schedule and complete training and notes per current guidelines.

 

5.      If upon receiving the email, the covering VSSC determines that the original Agent who terminated the ticket is out of the office or is no longer on the VSSC Program, Agent will send an email to NexusAdminSupport@corp.ds.fedex.com to help Un-terminate the PID.

 

6.      Once the PID has been Un-terminated by the Nexus Admin Support Team, VSSC will extract request from within their workload (note: Request will already be in VSSC workload as the original PID was terminated by VSSC).

 

 

7.      Schedule and complete training and notes per current guidelines.
