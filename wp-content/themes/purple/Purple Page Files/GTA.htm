The purpose of this process is to guide the VSSC Team when to use the GoToAssist (GTA) application in working with FedEx Customers to maximize training effectiveness.

Overview

GoToAssist™ is a Web-based technology that enables representatives to deliver live Customer Training anywhere in real time. GTA makes it easy for Agents to quickly and efficiently install applications or guide the Customer more effectively to navigate the applicable screens, which in turn makes a memorable customer experience every time. With this technology, a representative has the ability to:

  • Communicate with the customer via a chat box
  • View the customer’s desktop
  • Use the draw, highlight and laser pointer tools on the shared desktop
  • Push Web pages
  • Transfer files
  • Transfer a session to an escalation desk (if necessary)
  • Invite a co-worker to assist you or learn from a session

When to use GoToAssist:

As a general guideline, GTA should never be used as the first option when interacting with the Customer. Rather, it should be considered only after the VSSC attempts to walk-the-customer-through a training verbally over the phone, and runs into an obstacle that can best be overcome by using GTA to enhance the Customer’s experience with our application(s).

Examples of scenarios when obstacles might warrant use of GTA:

  • Database and File conversion assistance
  • Simple trouble shooting
  • Hold file setup
  • Template configuration
  • Custom report configuration
  • When there is a language barrier
  • When the customer is technically inexperienced
  • International Shipping
  • Assisting Customer with setting preferences.

Note: Usage of GTA is not limited to the above examples, and the application may be utilized for other scenarios indentified by the VSSC.

GTA Guidelines

  • Always tell the customer that you are planning to use the application & ask their permission prior to using it to access their system.
  • Let the customer control their own system, set up their own defaults, and make their own selections. Do not ‘take the controls’ and do it for them.
  • Let them determine where to save files
  • Let the customer create folders/ files or do any manipulating of their system when necessary
  • Never delete files from customer’s system
  • Never make changes to their operating system
  • Make sure that you end the session when the call is complete
  • Always remember to disconnect the application once your session with the Customer has been completed.
  • Per Quality Guidelines, Nexus comments must indicate that GTA was utilized and the reason for it’s use.
